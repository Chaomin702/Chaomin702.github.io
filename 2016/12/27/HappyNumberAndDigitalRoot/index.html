Happy Number And Digital Root

发表于 2016-12-27 | | 阅读次数

在LeetCode上遇到两个有趣的题目，写一下其背后的数学原理。

ADD Digital

Given a non-negative integer num, repeatedly add all its digits until the result has only one digit.
For example:
Given num = 38, the process is like: 3 + 8 = 11, 1 + 1 = 2. Since 2 has only one digit, return it.

Follow up:
Could you do it without any loop/recursion in O(1) runtime?

题目很简单，最简单的解法就如题目所述，不断地将num各位相加，最终得到一个个位数。问题是最终一定会得到一个个位数吗？

我们用$S(n)$来表示数字$n$的各数位之和，假设$n$有$m$位($m>1$)，$n=d_1+10d_2+\dots +10^{m-1}d_m$，那么$S(n)=d_1+d_2+\dots +d_m$，显然$S(n)<n$，除非$d_2,d_3,\dots,d_m=0$，也就是$n$为个位数。所以说对n不断地进行数位求和，n一定会减小至个位数。

事实上这里存在$O(1)$时间的解法，ADD Digital得到的数字称为digital root，digital root存在一条重要性质：

记$dr(n)$为数字n的digital root，那么


$$
dr(n) = \left{\begin{array}{}
0 &\text{if } n=0,\
9 &\text{if } n \neq 0,n\equiv 0(\text{mod }9),\
n \text{ mod } 9 &\text{if } n\not\equiv 0 (\text{mod }9)
\end{array}\right.
$$
简单点写就是$dr(n)=1+((n-1)\text{ mod }9)​$。

这条性质基于这样一个事实：$a\cdot100\equiv a\cdot10\equiv a \text{ mod }9$。

假设有一个三位数$n$，则$n = a\cdot 10^2+b\cdot10^1+c$，那么
$$
dr(n) \equiv a\cdot10^2+b\cdot10^1+c \equiv a+b+c \;(\text{mod } 9)
$$
从而得到结论：数字$n$与它的digital root对模9同余！

Happy Number

Write an algorithm to determine if a number is “happy”.

A happy number is a number defined by the following process: Starting with any positive integer, replace the number by the sum of the squares of its digits, and repeat the process until the number equals 1 (where it will stay), or it loops endlessly in a cycle which does not include 1. Those numbers for which this process ends in 1 are happy numbers.

Example: 19 is a happy number

  • $1^2 + 9^2 = 82$
  • $8^2 + 2^2 = 68$
  • $6^2 + 8^2 = 10$
  • $1^2 + 0^2 + 0^2 = 1$

将Happy Number的各数位不断地求平方和，最终会得到1，而非Happy Number则会进入一个死循环$4, 16, 37, 58, 89, 145, 42, 20, 4, \dots$

对于一个数$n$，假如它有$m$位，各数位平方和最多为$9^2m$，如果$m \geq 4$，那么$m\geq10^{m-1}>81m$，多求几次平方和，一定会减小为一个三位数，而最大的三位数999的各数位平方和为243，也就是$[244,999]->[1,243]$，多推演几次这样的情况，会发现三位数的各数位平方和也就单调递减的，最终会变为两位数。当然两位数就没有这样的单调递减特性了，需要更复杂的证明，这里略过。。

这道题说白了就是检测环路的问题，常见的单链表环路检测，或者hash表，或者直接去检测有没有4这个数字。
